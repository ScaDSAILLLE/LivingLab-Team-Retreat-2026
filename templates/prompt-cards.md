# Prompt Cards

## Understand Nanobot
```text
Explain this Nanobot setup for a mixed Living Lab team.
Cover:
1. What already works,
2. where configuration likely lives,
3. how the WebUI is started,
4. what would be needed for a public demonstrator,
5. what secrets or local configs must not be read or committed.
Keep it short and concrete.
```

## Keep Feature Work Small
```text
We have limited retreat time.
Suggest the smallest visible improvement for this feature track.
Separate tasks for developers, researchers, PR/accessibility, HPC/deployment, and documentation.
Avoid large architecture changes.
```

## Write for Non-Coders
```text
Write instructions for a person who does not program.
Explain:
1. what the demo does,
2. how to start or access it,
3. how to use it,
4. what to do if it fails.
Use clear, simple language.
```

## Safe Repo Check
```text
Check the repository state safely.
Show git status and changed files.
Do not read .env files or secret-looking configs.
Warn me before any file that might contain credentials is opened.
```
