# Welcome, Framing, and Day Objective

**Time:** 10:00-10:15

## Purpose
Create a shared reason for the day: demonstrators are not just technical artifacts, but tools for science communication, transfer, public engagement, and internal learning.

## Key Question
How can building one shared demonstrator help us understand each other's perspectives, responsibilities, and constraints?

## Objectives
- Establish the retreat as a learning, collaboration, and build format.
- Make the participating perspectives visible from the start: Living Lab, PR/communications, HPC/deployment, development, research, and non-coding roles.
- Define what a successful day should produce: a working baseline, reusable material, clearer responsibilities, and concrete next steps.

## Framing Points
- The demonstrator is the shared working object for the day, not the only goal.
- Scaddy-V2 is used as a concrete case to explore how the team can build, explain, operate, and maintain AI demonstrators together.
- opencode and Nanobot are part of the learning setup: they should support both coding and non-coding work.
- Contributions include implementation, testing, documentation, accessibility review, story, prompts, deployment thinking, and scientific assessment.

## Discussion Questions
- What does each team need from a demonstrator to consider it useful?
- Which responsibilities are often invisible to other sub-teams?
- Where do communication quality, technical reliability, scientific accuracy, and operational constraints meet?
- What should we learn today that is useful beyond this specific prototype?

## Moderation Notes
- Keep this short; the goal is alignment, not a full discussion.
- Name all relevant roles early so non-coding contributions are legitimate from the beginning.
- Avoid over-framing the technical solution; the next blocks create the shared understanding step by step.
- Capture one sentence for "success today" before moving on.

## Expected Output
- Shared statement of what a successful day looks like.
- Agreement that the workshop is both a build session and a perspective-exchange exercise.
