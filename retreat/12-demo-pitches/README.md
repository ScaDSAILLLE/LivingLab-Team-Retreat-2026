# Demo Pitches

**Time:** 15:50-16:10

## Purpose
Make the results visible, compare the value of the different feature extensions, and share what the team learned about building a public AI demonstrator with mixed perspectives.

## Key Question
What did we build or assess, what became clearer today, and what remains difficult before this could become a reliable Living Lab demonstrator?

## Objectives
- Show one concrete result, test, or integration assessment per active track.
- Compare technical value, public-demo value, accessibility, communication quality, and operational feasibility.
- Make limitations and next steps visible without overselling the prototype.
- Surface learnings across PR, Living Lab, HPC/deployment, development, research, and non-coding perspectives.

## Pitch Structure
Each group should use the same compact structure:

1. Initial idea.
2. What we used from Nanobot, opencode, templates, infrastructure, or existing material.
3. What we changed, configured, documented, tested, or assessed.
4. Live result, screenshot, fallback evidence, or integration decision.
5. Main limitation or risk.
6. One next step.
7. One learning about another team's perspective.

## Timeframe(s)
- **Maximum 5 minutes per group.**
- Prefer a working slice or clear assessment over a broad explanation.
- Stop before deep technical debugging; unresolved issues belong in follow-up notes.

## Audience Questions
- What would a visitor understand from this demo?
- What would PR or Living Lab need before showing it publicly?
- What would an operator need to keep this running?
- What should researchers or developers verify next?
- What should be reused in future demonstrators?

## Comparison Lens
Use the pitches to compare tracks along practical criteria:

| Lens | Guiding Question |
|---|---|
| Visitor value | Does this make Scaddy more useful or understandable? |
| Accessibility and wording | Is the interaction clearer or more inclusive? |
| Technical feasibility | Can this be maintained beyond the workshop? |
| Operations | What does deployment or public access require? |
| Follow-up value | Is the next step obvious and worth doing? |

## Expected Output
- Concise track demos or integration assessments.
- Shared comparison of technical, communication, design, and operations value.
- Clear limitations and next steps for each track.
- Input for the SWOT and closing block.

## Moderation Notes
- Keep pitches short and comparable.
- Encourage honest limitations; the pitch is not a sales demo.
- Ask for evidence: live result, screenshot, log, note, decision, or documented test.
- Make sure non-coding contributions are visible.
- Capture follow-up items immediately for the closing block.
