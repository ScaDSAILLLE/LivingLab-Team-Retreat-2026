# From Nanobot to Scaddy-V2

**Time:** 10:30-10:45

## Purpose
Translate the first Nanobot experience into a small, realistic Living Lab demonstrator scope for the day.

## Key Question
Which Nanobot capabilities can become a convincing Scaddy-V2 demonstrator today, and what must stay out of scope?

## Objectives
- Compare Nanobot's existing agent capabilities with the Scaddy-V1 reference.
- Identify the smallest demonstrator scope that still feels useful for a Living Lab setting.
- Separate today's buildable work from ideas that require later architecture, content, or deployment effort.

## Reference Context
- Scaddy-V1 is a multimodal conversational guide for the Living Lab.
- Visitors can ask questions, use speech, show images, and receive explanations in German or English.
- Scaddy combines interaction design, Living Lab knowledge, speech, vision, retrieval, and public-facing deployment.
- This retreat uses Nanobot as the starting point for a smaller Scaddy-V2-style prototype.

## Comparison Lens
Use the Nanobot exploration from the previous block and compare it against the Scaddy idea:

| Dimension | What to Look For |
|---|---|
| Interaction | What works in Nanobot WebUI, and what would confuse public visitors? |
| Knowledge | What Living Lab content would Scaddy need to explain? |
| Audience | How should answers differ for pupils, researchers, industry guests, or families? |
| Vision | What would a meaningful image-based demo show? |
| Voice | What is needed for speech to be useful rather than distracting? |
| Operations | What must be stable, documented, and secure for a public demo? |

## Scope Questions
- What is the smallest demo that still communicates the Scaddy idea?
- Which Scaddy-V1-inspired features are essential, optional, or out of scope today?
- Which parts are technical work, and which are content, design, communication, or operations work?
- Which idea is attractive but too large for this retreat?

## Expected Output
- A short Scaddy-V2 demonstrator scope for the day.
- A list of features or ideas parked for later work.
- Initial criteria for what the afternoon feature tracks should demonstrate.

## Moderation Notes
- Keep this as a scoping block, not a solution-design debate.
- Use the previous Nanobot observations as evidence.
- Actively separate "valuable later" from "buildable today".
- Make sure non-technical requirements are captured as real scope, not as decoration.
