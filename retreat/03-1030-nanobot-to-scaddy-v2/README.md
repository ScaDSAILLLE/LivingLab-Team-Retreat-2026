# 10:30-10:45 - From Nanobot to Scaddy-V2

## Purpose
Translate the first Nanobot experience into a small and realistic Living Lab demonstrator goal.

## Key Question
What is already available, what is missing, and what would make this convincing as a Living Lab demonstrator?

## Reference Context
- Scaddy-V1 is a multimodal conversational guide for the Living Lab.
- Visitors can ask questions, use speech, show images, and receive explanations in German or English.
- Scaddy combines STT, TTS, vision processing, retrieval, and a public-facing tablet interface.
- For this retreat, Nanobot is the foundation and comparison object; the goal is a smaller Scaddy-V2-style prototype.

## Comparison Dimensions
- Public-facing interaction versus developer/operator WebUI.
- Living Lab knowledge and audience-appropriate explanations.
- Vision input and visual exhibit explanations.
- Voice input/output and transcript visibility.
- Accessibility and busy exhibition environment.
- Deployment, persistence, logs, updates, and secrets.

## Discussion Questions
- What is the smallest demo that still communicates the Scaddy idea?
- Which Scaddy-V1 features are essential, optional, or out of scope for one day?
- Which features are technical work, and which are content/design/communication work?
- What would different visitor groups need: pupils, researchers, industry guests, families?

## Expected Output
- A short Scaddy-V2 demonstrator scope for the day.
- A parking lot for ideas that are valuable but too large.
