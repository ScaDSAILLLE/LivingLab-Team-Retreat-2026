# 10:15-10:30 - Explore Nanobot and WebUI

## Purpose
Create a quick first success with the actual foundation for the day.

## Key Question
What can Nanobot already do, and what feels immediately useful or surprising?

## Objectives
- Open Nanobot WebUI and test basic interaction.
- Identify what Nanobot already provides before inventing custom UI work.
- Surface first impressions from technical and non-technical participants.

## Facilitator Instructions
- Start from the prepared sandbox or RPi target if available.
- Show WebUI first; avoid chat-app integrations and messenger tokens.
- Keep the block short and concrete.

## Test Tasks
- Ask Nanobot to explain what it is in one paragraph.
- Ask it to summarize the retreat goal for a non-technical visitor.
- Ask it what would be needed to turn this into a Living Lab demonstrator.
- If file upload/API support is available, test one harmless public document or image.

## Reflection Questions
- Which parts already feel like a demonstrator?
- Which parts are too agent/tool-oriented for public visitors?
- What would PR, accessibility, HPC, research, and development each worry about?
- What should we not customize today because Nanobot already gives us enough?

## Expected Output
- List of useful existing Nanobot capabilities.
- List of gaps or risks for Scaddy-V2.
